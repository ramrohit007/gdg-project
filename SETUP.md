# Quick Setup Guide

## Step 1: Database Setup

1. Install PostgreSQL if not already installed
2. Create a new database:
```sql
CREATE DATABASE student_performance;
```

3. Update the `DATABASE_URL` in `backend/.env`:
```
DATABASE_URL=postgresql://username:password@localhost:5432/student_performance
```

## Step 2: Backend Setup

```bash
cd backend

# Install Python dependencies
pip install -r requirements.txt

# Create .env file (copy from .env.example and update values)
# Windows:
copy .env.example .env
# Linux/Mac:
cp .env.example .env

# Initialize database with demo accounts
python init_db.py

# Start the backend server
uvicorn main:app --reload
```

The backend will run on `http://localhost:8000`

## Step 3: Frontend Setup

Open a new terminal:

```bash
cd frontend

# Install dependencies
npm install

# Start the development server
npm run dev
```

The frontend will run on `http://localhost:3000`

## Step 4: Access the Application

1. Open your browser and go to `http://localhost:3000`
2. Login with demo accounts:
   - **Teacher**: username: `teacher`, password: `teacher123`
   - **Students**: username: `student1`/`student2`/`student3`, password: `student123`

## Important Notes

- Make sure PostgreSQL is running before starting the backend
- The OpenAI API key is optional - the system will use mock responses if not provided
- Login codes generated by teachers expire after 1 hour
- PDF files should contain extractable text (not scanned images)

## Troubleshooting

### Database Connection Error
- Verify PostgreSQL is running
- Check database credentials in `.env` file
- Ensure the database `student_performance` exists

### Port Already in Use
- Backend: Change port in `uvicorn main:app --reload --port 8001`
- Frontend: Change port in `vite.config.ts` and update proxy target

### CORS Errors
- Ensure backend CORS settings include your frontend URL
- Check that both servers are running
